出海体检官：详细说明

LaunchFit AI 把“商品能不能出海、应该查什么、哪里会卡审”转成可执行的信息搜集、核验和报告流程。它不是离线政策百科，而是一套面向跨境商品上架前决策的结构化审核框架。

核心原则：用户必须提供原产地和一个或多个目标市场。系统按目的地拆分审核路径，先生成最合适的信息渠道和核验任务，再输出风险、缺口和行动建议。

1. 输入范围

输入说明为什么重要
原产地生产、组装或出口地影响出口控制、COO、制造商/出口商文件、标签原产地一致性
目标市场支持多个国家或区域每个市场的监管、进口、标签、平台规则不同，必须拆分审核
平台和类目Amazon、TikTok Shop、Temu、Lazada、Tmall Global 等决定准入、禁限售、类目资质、内容和履约要求
商品和包装名称、规格、成分/材料、宣称、标签、警示、认证标识影响标签重印、功效宣称、认证标识使用和消费者信任
证书和授权检测报告、证书、品牌授权、商标、企业资料用于判断是否过期、错配、授权范围不足或需要外部核验
对标和物流竞品截图/链接、价格、渠道、物流路线、报价用于判断价格带、包装惯例、履约风险和利润压力

2. 输出结构

market_reviews[]

每个目的地一份独立审核轨道，包含匹配规则包、要求、发现、缺口、渠道和任务。

source_candidates[]

系统建议优先查询的信息渠道，覆盖官方来源、专业来源和用户自有渠道。

research_tasks[]

每个渠道对应的核验任务：查什么、为什么、优先级、证据字段、刷新周期。

report / memo

结构化 JSON 和 Markdown 备忘录，方便服务商、运营、合规和供应商协作。

渠道类型要查什么默认层级

4. 证据层级

层级来源用途
T1平台官方政策、监管机构、政府/海关/法院/注册数据库可支持关键决策
T2认证机构、认可机构、标准机构、实验室资质查询可支持证书和技术路线核验
T3律所、报关行、合规顾问、行业协会用于解释或补充，不替代官方来源
T4申请人文件、供应商陈述、截图、邮件、合同、用户搜索渠道证明“提交了什么”，不能默认证明事实为真
T5社媒、论坛、非正式视频、不可核验说法只能做市场信号，不能做资质依据

5. 决策状态

状态含义
approve当前材料和核验结果支持推进。
conditional_approve可推进，但需要完成边界清楚的低/中风险补正。
request_more_info关键信息或材料缺失，暂不能判断。
reject已确认禁售、严重不合规、无权销售、材料失效且不可补正。
escalate_human疑似造假、制裁/出口管制、法律歧义或官方来源冲突。

6. 为什么这样更好用

  • 先问最关键的两个边界：原产地和目标市场。
  • 多目的地自动拆分，不把不同司法辖区规则混在一起。
  • 先产出“应该查哪里”和“查什么”，再输出结论，避免泛泛建议。
  • 允许接入用户自己的搜索渠道，但通过证据层级防止误当官方事实。
  • 输出可复核、可交接、可补件的结构，而不是一次性聊天建议。
